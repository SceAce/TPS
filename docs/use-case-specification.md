# TPS 用例说明书

> 文档说明：本文件细化系统用例、参与者与前后置条件。


**版本：** v1.0  
**日期：** 2026-05-05  
**用途：** 作为《软件工程》课程“需求分析”阶段的标准化用例说明文档

---

## 1. 角色定义

- 普通用户：已注册并登录的校园用户。
- 买家：在特定交易场景中发起购买的一方。
- 卖家：在特定交易场景中发布商品的一方。
- 管理员：拥有平台治理权限的后台管理角色。

---

## 2. 用例 UC-01 注册/登录

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-01 |
| 用例名称 | 注册/登录 |
| 参与者 | 普通用户 |
| 前置条件 | 用户未登录；系统可访问 |
| 后置条件 | 登录成功后获得 token 和角色信息 |
| 触发条件 | 用户进入登录页或注册页 |

主成功场景：

1. 用户输入手机号、验证码/密码及必要资料。
2. 系统校验输入合法性。
3. 系统完成注册或登录。
4. 系统返回 token、用户编号、角色等信息。
5. 客户端进入对应主界面。

备选场景：

- 2a. 输入格式错误：系统提示用户修正。
- 3a. 验证码或密码错误：系统提示登录失败。
- 3b. 账号被封禁：系统拒绝登录。

## 3. 用例 UC-02 发布商品

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-02 |
| 用例名称 | 发布商品 |
| 参与者 | 卖家 |
| 前置条件 | 用户已登录 |
| 后置条件 | 商品保存成功并进入在售状态 |
| 触发条件 | 用户点击“发布商品” |

主成功场景：

1. 卖家进入发布页面。
2. 填写标题、描述、价格、分类、成色、所在地。
3. 上传商品图片。
4. 提交发布请求。
5. 系统校验字段与图片合法性。
6. 系统保存商品和图片信息。
7. 系统返回商品详情。

备选场景：

- 5a. 缺少必填项：提示补全信息。
- 5b. 图片不合法：拒绝上传并提示原因。

## 7. 用例 UC-06 创建订单

| 项目 | 内容 |
|---|---|
| 用例编号 | UC-06 |
| 用例名称 | 创建订单 |
| 参与者 | 买家 |
| 前置条件 | 用户已登录；商品处于在售状态 |
| 后置条件 | 成功生成订单，商品进入不可重复购买状态 |
| 触发条件 | 买家发起购买 |

主成功场景：

1. 买家在商品详情页发起下单。
2. 系统校验商品状态、价格和买家身份。
3. 系统创建订单。
4. 系统将商品状态调整为下架锁定态。
5. 卖家收到订单通知。

备选场景：

- 2a. 商品已售出或已下架：拒绝创建。
- 2b. 买家购买自己商品：拒绝创建。
- 2c. 已存在进行中订单：拒绝创建。

|---|---|
| 用例编号 | UC-08 |
| 用例名称 | 申请退款 |
| 参与者 | 买家、卖家、管理员 |
| 前置条件 | 订单已付款或已发货 |
| 后置条件 | 订单进入退款中或已退款状态 |
| 触发条件 | 买家发起退款申请 |

主成功场景：

1. 买家提交退款申请和原因。
2. 系统将订单状态变更为退款中。
3. 卖家处理退款，或管理员介入审批。
4. 系统将订单变更为已退款。
5. 商品重新回到可售状态。
