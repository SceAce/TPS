# TPS 测试计划

> 文档说明：本文件说明测试目标、测试范围、方法与执行安排。


**版本：** v1.0  
**日期：** 2026-05-05

---

## 1. 测试目标

验证 TPS 的核心交易链路、权限控制、数据一致性和异常处理是否符合需求。

---

## 2. 测试范围

- 认证：注册、登录、刷新 token、登出。
- 商品：发布、编辑、下架、搜索、详情、举报。
- 收藏：收藏、取消收藏、状态查询。
- 消息：会话创建、消息收发、已读标记。
- 订单：创建、支付、发货、确认、退款、评价。
- 管理：用户管理、商品下架、举报处理、公告发布、反馈处理。

---

## 3. 测试策略

- 单元测试：服务层核心规则。
- 集成测试：Controller + Service + Repository。
- 接口测试：按 API 契约逐项验证。
- 回归测试：覆盖核心主链路。

---

## 4. 重点风险

- 重复下单。
- 重复评价。
- 越权查看消息和订单。
- 图片地址不可访问。
- token 过期后的恢复行为。

---

## 5. 测试通过标准

- 核心主链路全部通过：注册 -> 登录 -> 浏览 -> 发布 -> 聊天 -> 下单 -> 支付 -> 发货 -> 收货 -> 评价。
- 管理员链路通过：登录 -> 用户管理 -> 举报处理 -> 商品下架 -> 统计查看 -> 反馈回复。
- 所有权限相关接口返回符合约定的 JSON 结构。
- 订单、举报、评价、反馈相关状态流转符合设计说明。

---

## 6. 测试输出物

- 测试计划
- 测试用例表
- 测试执行记录
- 缺陷记录
- 验收结论
