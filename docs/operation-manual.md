# TPS 部署与运维手册

> 文档说明：本文件提供系统部署、运行与日常操作手册。


**版本：** v1.0  
**日期：** 2026-05-05

---

## 1. 启动前检查

- JDK 17 可用。
- MySQL 可用。
- Redis 可用。
- `img/` 目录存在且可写。

---

## 2. 后端启动

```bash
cd /home/source/myGtihub/TPS
./start-backend.sh
```

如需指定局域网地址：

```bash
TPS_LAN_IP=192.168.33.96 ./start-backend.sh
```

---

## 3. 后端验证

- `http://127.0.0.1:8080/swagger-ui.html`
- `http://127.0.0.1:8080/api/files/ping`

---

## 4. 图片访问

图片通过 `TPS_PUBLIC_BASE_URL` 转为绝对地址后返回给客户端，避免移动端访问相对路径失败。

---

## 5. 备份建议

- 数据库定期备份。
- `img/` 目录定期备份。
- 保留上一版本后端包。

---

## 6. 常见故障

- `401`：token 无效或过期。
- `403`：权限不足。
- 图片无法加载：检查 `TPS_PUBLIC_BASE_URL` 和上传目录。
- 消息无法接收：检查 WebSocket 地址和鉴权。

---

## 7. 管理员运维建议

- 定期检查异常用户与举报商品。
- 定期清理无效测试图片和测试数据。
